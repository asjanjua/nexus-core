# NexusAI V1 Pilot Onboarding Checklist

## 1) Sponsor and Scope
- Confirm buyer lane from `docs/USER_STRATEGY_AND_PIVOTS.md`:
  - evaluator/SME
  - SME self-serve
  - business/advisory
  - regulated enterprise
- Confirm executive sponsor (CEO/COO/CBO equivalent).
- Confirm named workflow reviewer or approver.
- Confirm 6-8 week pilot window.
- Lock first workflow target and outcomes: decision clarity, risk visibility, recommendation quality, cycle-time reduction, or rework reduction.

## 2) Workspace Setup
- Create tenant + workspace.
- Assign role mapping (CEO, COO, CBO/Strategy).
- Configure user allowlist.
- Attach readiness band, buyer lane, and first workflow target to workspace notes or profile.

## 3) Source Bundle (Docs + Comms)
- Connect Google Drive or SharePoint pilot folders.
- Connect Slack (mention-based) channels/threads.
- Validate upload ingestion path for PDF/DOCX/PPTX/XLSX.
- Confirm which sources prove the first workflow target.

## 4) Governance Policy
- Set quarantine threshold (default 0.55).
- Require approval for recommendation promotion.
- Confirm restricted-data masking for Slack summaries.
- Confirm no autonomous source-system writeback, external communication, payment, filing, HR action, legal commitment, or financial commitment in V1.
- Confirm approval boundary for the selected first workflow.

## 5) Validation
- Run ingestion status check.
- Generate first CEO/COO/CBO dashboards.
- Produce first Weekly Executive Brief and Decision Memo.
- Run workflow scorer and record sponsor-approved first workflow.
- Capture baseline manual time, Nexus-assisted time target, and shadow ROI metric.
