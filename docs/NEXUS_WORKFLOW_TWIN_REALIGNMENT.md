# NexusAI Workflow Twin Realignment

## Status
Decision note, 2026-05-31.

## Core Decision
NexusAI should be framed as a governed intelligence operating layer for high-stakes professional workflows, not as a generic autonomous-company platform.

This realignment now connects to the user strategy in `docs/USER_STRATEGY_AND_PIVOTS.md`: readiness assessment -> buyer lane -> signup/onboarding -> first workflow pilot -> governed value proof.

The product should borrow useful ExO-style mechanics, but translate them into practical enterprise language:

| Avoid | Use instead |
| --- | --- |
| MTP protocol | Strategic Mandate, Operating Doctrine, Policy Guardrails |
| Digital twin at the edge | Parallel Workflow Pilot |
| Fiduciary wedge | Human Approval Layer |
| Recursive self-improvement | Feedback and Learning Loop |
| Intelligence stack | Decision Workflow Engine |
| Fully autonomous company | Governed human-approved workflows |

Do not use broad "100x", workforce-replacement, or fully autonomous organization claims in client-facing copy.

## Build Sequence
The governance sequence remains first because regulated buyers need it before they can trust the product:

1. Finish U2 Agent Control Profiles.
2. Build U3 per-agent logs and rollback-ready output history.
3. Build U4 learning-signal capture.

In parallel, the workflow-twin roadmap should start broad before becoming industry-specific:

1. Decision & Action Twin.
2. Workflow Twin Scorer and client workflow selection.
3. Ops Review Twin.
4. Specialized templates: Proposal/SOW, Regulatory Response, Agreement Review, Due Diligence, Board Memo, PMO Tracking.

## First Workflow Twin
The first universal twin is the Decision & Action Twin.

Inputs:
- Approved evidence.
- Meeting notes.
- Slack/Teams messages.
- Uploaded documents.
- Dashboard signals.
- Existing decisions and recommendations.

Outputs:
- Decisions.
- Action items.
- Owners.
- Risks.
- Blockers.
- Recommendations.
- Evidence references.
- Freshness and confidence.

Human approval remains required before an action item, recommendation, decision, or external output becomes canonical.

## Buyer-Lane Workflow Strategy

| Buyer lane | First workflow strategy | Good first candidates | Avoid first |
| --- | --- | --- | --- |
| Evaluator / SME | Prove usefulness in one owner/operator loop with minimal setup. | Weekly owner brief, cash/runway review, customer issue review, ops blocker report. | Complex governance, multi-team workflow maps, procurement-heavy rollout. |
| SME self-serve | Convert uploaded sources into plain-language operating clarity. | Weekly synthesis, sales pipeline review, hiring or vendor blocker report. | Enterprise terminology, deep role taxonomy, custom agent governance. |
| Business / advisory | Turn scoping into a sponsor-ready paid pilot. | Proposal pipeline review, board-pack variance summary, project delay report, executive risk brief. | Vague "AI transformation" pilots without named owner or reviewer. |
| Regulated enterprise | Start where governance can be demonstrated before automation is discussed. | Regulatory issue tracker summary, supplier risk review, control evidence pack, board risk brief. | Autonomous external action, filings, payments, HR/legal/financial commitments, source-system writeback. |

Every workflow pilot should name the sponsor, reviewer, evidence bundle, approval boundary, and shadow ROI metric before it enters paid scope.

## Governance Boundary
Every workflow twin must obey Agent Control Profiles before evidence enters model context.

In V1, NexusAI must not autonomously:
- Send emails or messages.
- Submit filings.
- Make payments.
- Change contracts.
- Contact regulators.
- Take HR actions.
- Make legal or financial commitments.
- Write back to source systems.

## Product Positioning
Recommended positioning:

> NexusAI helps expert teams move from manual coordination to governed intelligence. It converts scattered documents, meetings, messages, and institutional knowledge into decisions, actions, risks, recommendations, and reusable company memory, with agent passports, approval queues, audit logs, and human accountability built in.
