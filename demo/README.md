# Nexus Demo

This folder shows the shape of Nexus outputs without requiring a live customer workspace.

The demo story should match the current strategy: readiness -> buyer lane -> first workflow pilot -> governed value proof.

## What To Look At
- `sample-executive-brief.md` for a readable executive output
- the install command in the main README for the fastest first try
- `/knowledge` in Mission Control for the v0.25.0 Knowledge Workspace when running the app

## Example Use
1. Install Nexus.
2. Scaffold a workspace.
3. Point it at a few source folders.
4. Pick one workflow worth proving.
5. Generate a brief and see the evidence trail.
6. Capture the reusable context in Knowledge Workspace notes.

The point of the demo is to make the product feel tangible before someone installs it.
